# Code Graph Design Archive

> **Status: NOT IMPLEMENTED.** These documents describe capabilities that were
> never wired into the Safety OS runtime. Archived 2026-06 per the
> **Truth-in-Documentation** principle (documented capability must equal
> implemented capability).

## Scope

This directory holds historical design documents for two distinct, unimplemented
initiatives:

1. **CodeGraph MCP** — AST-based codebase exploration server. The codegraph MCP
   was removed from the runtime on **2026-06-16**; its entry never appeared in
   `.mcp.json`.
2. **Neo4j Knowledge Graph Traceability Model** — an aspirational design for
   linking evidence, workflows, and regulations via a Neo4j-backed graph.
   Neo4j was **never** wired into `.mcp.json` and was never deployed. The
   "Knowledge Graph Traceability Model" remained a design-only artifact.

## How regulatory traceability is actually achieved

Safety OS does **not** depend on a Neo4j graph for traceability. Regulatory
traceability is delivered by these live, implemented mechanisms:

- **`evidence-models/*.json`** — structured evidence record schemas (the live
  replacement for the aspirational `graph-schema.json`, which has been deleted).
- **Workflow `schema.yaml` `legal_basis` fields** — every workflow carries an
  explicit legal-basis reference to Korean statute articles (e.g.,
  `산업안전보건법 (OSHA-KR) Art 36`, `중대재해처벌법 (SAPA)`).
- **`regulations/KR/legal-glossary.yaml`** — the single source of truth (SSOT)
  for statute-name to English-gloss mapping (`source_mcp: mcp_kr_legislation`).

## Active MCP servers (runtime)

The active `.mcp.json` registers exactly these servers — none of them is a graph
database:

- `k_skill`
- `legalize_kr`
- `mcp_kr_legislation` (Korean legislation — source for the legal glossary SSOT)

## Retention rationale

These documents are preserved as **design-history / audit-traceability
artifacts**. They record the design intent, the evaluation that led to
non-adoption, and the lineage of the current evidence/glossary architecture. Do
not treat them as a specification of implemented behavior.
