### Why co-architect has `.agents/skills/` but Safety OS doesn't need it

- **co-architect**: `sync` and `meeting` live ONLY in `.agents/skills/` (not in SSOT `skills/`). Phase 2 of `sync-skills.ts` distributes these shortcut skills back to `.claude/skills/` and `.gemini/skills/`.
- **Safety OS**: Both `sync` and `meeting-facilitation` are proper SSOT skills in `skills/`. Phase 1 of `sync-skills.ts` already distributes them to all platform directories. No shortcut layer needed.

### Structural difference: project-review has no command file

Both co-architect and Safety OS intentionally do NOT create `.claude/commands/project-review.md` or `.gemini/commands/project-review.md`. The `project-review` skill is triggered by skill-name matching (e.g., `/project-review` or natural language), not via a slash command stub. This is by design — it's a heavyweight process skill that doesn't benefit from the command file shortcut pattern.
