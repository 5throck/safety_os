## D-01 후속: legal_basis 확정 (사용자 승인)
사용자 요청으로 LOTO 법령 조회를 더 꼼꼼히 재시도 — `mcp_kr_legislation`(get_current_law, get_compliance_guide), `legalize_kr`(parse_law_structure ×3 변형, search_admrule, find_references), `k_skill`(search_osha_regulations) 총 3개 법령 MCP, 7회 시도. 결론: **"산업안전보건기준에관한규칙"(LOTO 세부조항이 실제로 담긴 하위규칙)은 이 프로젝트에 연결된 어떤 법령 MCP에도 색인되어 있지 않음** — 모법(산업안전보건법) 전체 조문(1,294줄)을 직접 확보해 전문 검색해도 잠금/기동방지/운전정지 키워드 0건. 제38조가 "기계·기구에 의한 위험 방지조치"를 선언하고 세부사항을 "고용노동부령"(하위규칙)에 위임하는 구조만 확인됨 — 이는 도구 커버리지의 한계이지 추가 시도로 해결 가능한 문제가 아님.

## gh pr create 401 반복 오류 근본원인 규명 및 수정 (사용자 승인)
이번 세션 내내 `bun scripts/dev-sync.ts` 실행 시 `gh pr create` 단계에서 반복적으로 "HTTP 401: Bad credentials"가 발생했으나(`gh auth status`는 항상 정상, 직접 Bash로 `gh pr create` 호출하면 항상 성공) — 근본원인을 규명:

- **수정**: `scripts/dev-sync.ts`(v1.3.0, PR body/gh 로직 구간)에 `ghEnv`(process.env에서 `GITHUB_TOKEN`/`GH_TOKEN` 제거한 복사본) 정의 후, 3곳의 `gh pr create` 호출에 `.env(ghEnv)`를 추가 — `.env` 파일 자체는 건드리지 않고(다른 용도로 필요하므로), `gh` 호출에만 국한해 이 두 환경변수가 새어들어가지 않도록 격리.
