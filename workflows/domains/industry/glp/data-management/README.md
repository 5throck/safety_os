# data-management Workflow

## 1. Objective
Manage GLP raw data with full ALCOA+ integrity per OECD GLP Section 9, 10.

## 2. Applicability
All raw data generated during GLP studies.

## 3. ALCOA+ Principles (apply `skills/domains/glp/glp-data-integrity-checker/`)
- **A**ttributable: Each entry traceable to specific individual
- **L**egible: Permanent, indelible records
- **C**ontemporaneous: Recorded at time of observation
- **O**riginal: First recording (or certified true copy)
- **A**ccurate: Error-free, verifiable
- **+ Complete**: All data including repeats/reanalyses
- **+ Consistent**: Chronological, dated
- **+ Enduring: On durable medium**
- **+ Available: Accessible for review**

## 4. Workflow Steps
1. **Raw data capture**: Direct electronic capture preferred.
2. **Audit trail**: All changes tracked with reason, signature.
3. **Electronic records**: 21 CFR Part 11 / EU Annex 11 compliance.
4. **Archive**: Secure, indexed retention (minimum 10 years).
5. **Retrieval**: Available for regulatory inspection.

## 5. Evidence Record
Generate `glp-data-record.json` with retention metadata.
