# benefit-risk-assessment Workflow

## 1. Objective
Conduct ongoing benefit-risk assessment per EU GVP Module 12.

## 2. Frameworks
- **PrOACT-URL**: Problem, Objectives, Alternatives, Consequences, Trade-offs, Uncertainty, Risk, Links
- **BRAT**: Benefit-Risk Action Team framework
- **MCDA**: Multi-Criteria Decision Analysis (quantitative)

## 3. Workflow Steps
1. **Define framework**: Choose methodology (apply `skills/domains/gvp/benefit-risk-assessor/`).
2. **Benefit identification**: Efficacy endpoints from pivotal trials + RWE.
3. **Risk identification**: From PBRER, ICSR database.
4. **Value tree**: Hierarchical benefit/risk model.
5. **Weighting**: Stakeholder input (patients, clinicians).
6. **Score integration**: Quantitative or qualitative.
7. **Decision**: Maintain / Restrict / Withdraw.
8. **Documentation**: Benefit-risk report for regulatory submission.

## 4. Evidence Record
Generate `gvp-br-record.json` with framework, scores, decision.
